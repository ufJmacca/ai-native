# Todo API

## Problem

We need a small but production-shaped JSON API for managing todo items. The API should support creating, listing, updating, completing, and deleting todo items.

## Users

- Individual developers evaluating the AI-native template.
- QA engineers who need repeatable acceptance criteria.

## Constraints

- Keep the first implementation intentionally small.
- Use tests before implementation changes.
- Document any assumptions directly in the PRD.

## Acceptance Criteria

- Exposes CRUD endpoints for todo items.
- Persists todo items in a lightweight local store.
- Includes automated tests for the API behavior.
- Produces a README section explaining how to run the sample app.
